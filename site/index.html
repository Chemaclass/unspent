What it does

Every unit of value is an unspent output. Spending consumes outputs and creates new ones — so double-spends are structurally impossible and the full history is always auditable.

🔒

Double-spend safe

Outputs can be spent exactly once. Invalid transactions are rejected before they mutate the ledger.

🧾

Immutable audit trail

Domain objects are readonly. Every output tracks what created and spent it.

💾

Pluggable persistence

In-memory for tests, SQLite for production — or implement the repository port for any backend.

🗝️

Rich lock types

Owner, Ed25519 public key, M-of-N multisig, time locks, and hash locks — or bring your own.

📡

Events & logging

PSR-14 event dispatching and PSR-3 logging via composable decorators.

📊

Selection & analytics

Coin-selection strategies (FIFO, largest/smallest-first, …) and UTXO analytics out of the box.

Fluent by design

Mint value, move it between owners, and query balances — with a chainable API.

use Chemaclass\Unspent\Ledger;

$ledger = Ledger::inMemory();

$ledger->credit('alice', 100)          // mint 100 to alice
       ->transfer('alice', 'bob', 30); // alice → bob

$ledger->totalUnspentByOwner('bob');   // 30
$ledger->totalUnspentByOwner('alice'); // 70

// Validate before committing — no exceptions thrown
$ledger->canApply($tx); // null if valid, else the reason
